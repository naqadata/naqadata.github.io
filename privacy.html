Naqafin Privacy Policy

Effective date: June 16, 2026

Naqafin is an unofficial Roku client for Jellyfin-compatible media servers. The app lets you connect your Roku device to a media server that you configure. Naqafin does not provide media content, hosted accounts, subscriptions, or a hosted media service.

Summary

Naqafin does not sell personal information, display ads, or include third-party analytics SDKs. Naqafin does not operate a backend service that receives your media library, viewing activity, Jellyfin credentials, authentication tokens, crash reports, or diagnostics from the app.

The app communicates with the Jellyfin-compatible server that you choose. Your server controls the media library, user accounts, metadata, playback history, plugins, and related data available through the app. Roku may separately collect device, platform, usage, crash, or diagnostic information under Roku's own privacy policies.

Information Naqafin Handles

To work as a media client, Naqafin may handle the following information on your Roku device and in communications with your configured server:

  • Server connection details, such as the server URL you enter and saved server entries.
  • Jellyfin user information returned by your server, such as user ID, username, profile image, policy, and display preferences.
  • Authentication information for your Jellyfin server. Your password is used to authenticate with your server. If you choose to save credentials or stay signed in, Naqafin stores the server-issued authentication token and username in Roku on-device registry storage.
  • Media library information returned by your server, including titles, metadata, artwork URLs, library structure, playlists, favorites, watched status, resume position, subtitles, audio tracks, and playback options.
  • Playback session data needed for streaming, resume, transcoding, direct play, playlists, subtitle selection, playback progress, and server session reporting.
  • Roku device information needed for playback and server compatibility, such as device model, display mode, codec capabilities, locale, caption settings, and a Roku channel client identifier used as a device ID when communicating with your server.
  • App settings saved on the Roku device, such as playback, subtitle, library, appearance, and home screen preferences.

Where Information Is Stored

Naqafin stores app settings, saved server entries, saved usernames, and saved server authentication tokens in Roku on-device registry storage. This storage is local to the Roku app environment. Naqafin also may use temporary Roku storage for transient playback assets such as trickplay images or downloaded fallback fonts requested from your server.

Naqafin does not store your Jellyfin password after authentication. If you save your login, Naqafin stores the authentication token issued by your Jellyfin-compatible server, not the password itself.

Information Sent to Your Server

Naqafin sends requests to the Jellyfin-compatible server you configure. These requests may include your server authentication token, Jellyfin user ID, device/client information, device playback capabilities, playback progress, selected media IDs, search/filter choices, playlist actions, favorite and watched-state actions, subtitle actions, and other information needed for the feature you use.

If you use server-generated caption features, Naqafin requests caption sessions from your configured server and reads caption data from server-provided WebVTT endpoints. If your server is configured to use an optional remote caption worker, that server-side workflow is controlled by your server configuration. Naqafin talks to the Jellyfin server API and does not contact the caption worker directly.

Information Sent to Naqadata

Naqafin does not send media library data, viewing history, credentials, authentication tokens, analytics, telemetry, crash logs, or diagnostics to Naqadata from the Roku app.

If you contact Naqadata for support, submit a GitHub issue, send logs, or otherwise provide information voluntarily, Naqadata may receive and use that information to respond to your request, investigate issues, and improve the app.

Roku Platform Data

Naqafin runs on the Roku platform. Roku may collect information about your device, installed channels, channel usage, crashes, performance, diagnostics, and other platform activity under Roku's privacy policies and account settings. Naqafin also uses Roku platform features required for normal channel operation and performance signaling. Naqadata does not control Roku's data practices.

Third Parties

Naqafin does not include advertising networks, third-party analytics SDKs, or in-app purchases. The app does not sell or rent personal information.

Your configured Jellyfin-compatible server, server plugins, metadata providers, subtitle providers, remote workers, reverse proxies, hosting providers, and network services may have their own privacy and logging practices. Those systems are controlled by you or the server operator, not by Naqafin.

The policy pages for Naqafin may be hosted on GitHub Pages. GitHub may process access logs and related information under GitHub's own privacy practices.

Security

Naqafin supports connecting to servers over HTTPS and uses the Roku platform certificate store for HTTPS requests. You are responsible for configuring and securing your server, network, domain, certificates, user accounts, passwords, authentication tokens, and server plugins.

You should only connect Naqafin to servers you own, administer, or are authorized to access. If you no longer trust a Roku device, server, or account, revoke the relevant server sessions or tokens from your Jellyfin-compatible server and remove saved data from the Roku app or device.

Your Choices

  • You can choose which server to connect to.
  • You can choose whether to save your login where the app offers that option.
  • You can sign out or remove saved users to delete saved authentication tokens from the app's local Roku storage.
  • You can remove the channel or factory reset the Roku device to clear app data according to Roku platform behavior.
  • You can manage users, sessions, tokens, libraries, metadata, plugins, and playback history on your Jellyfin-compatible server.

Children

Naqafin is not directed to children. The app displays media and metadata from the server you configure. Parents and server administrators are responsible for configuring user access, parental controls, libraries, and content restrictions on their server and Roku account.
